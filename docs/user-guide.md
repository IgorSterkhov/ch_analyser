# Руководство пользователя

## Обзор интерфейса

Веб-интерфейс имеет два основных режима, переключаемых кнопками в шапке:

**All Servers** — дашборд мониторинга всех серверов:
```
+----------------------------------------------------------+
| Header: All Servers | Server Details | ⚙ | ? | user      |
+----------------------------------------------------------+
| Диски серверов (таблица + график)                         |
+----------------------------------------------------------+
| Таблицы сервера (таблица + график)                        |
+----------------------------------------------------------+
```

**Server Details** — детальный анализ конкретного сервера:
```
+----------------------------------------------------------+
| Header: All Servers | Server Details | ⚙ | ? | user      |
+-------------+--------------------------------------------+
|             | Информация о сервере: диск, использование   |
| Подключения +--------------------------------------------+
| (боковая    | Tables / Users / Text Logs                 |
|  панель)    |                  Детали таблицы (правая     |
|             |                  выдвижная панель):         |
|             |                  Колонки / История / Flow   |
+-------------+--------------------------------------------+
```

## Авторизация

При первом входе откроется страница логина. Введите имя пользователя и пароль, назначенные администратором. Доступны две роли:

- **admin** - полный доступ: просмотр данных, управление подключениями
- **user** - только просмотр данных по существующим подключениям

## Подключение к серверу

1. Перейдите в режим **Server Details** (кнопка в шапке)
2. Выберите подключение из выпадающего списка **Connection** в верхней части
3. При успешном подключении автоматически загрузится информация о сервере и список таблиц

Управление подключениями (добавление, редактирование, удаление) — через диалог настроек (⚙).

## Информация о сервере

После подключения в верхней части отображается:

- Имя подключения
- Для каждого диска: использованное / общее пространство, процент заполнения
- Цветовая индикация: зелёный (< 70%), жёлтый (70-90%), красный (> 90%)

## Дашборд All Servers

Страница мониторинга, доступная по кнопке «All Servers» в шапке. Показывает состояние всех подключённых серверов.

### Диски серверов

Верхняя секция отображает таблицу с информацией о дисках:

| Колонка | Описание |
|---------|----------|
| Server | Имя подключения |
| Used | Использованное пространство |
| % | Процент заполнения |
| Status | Цветовой индикатор: OK (зелёный), Warning (жёлтый), Critical (красный) |
| 📊 | Иконка `table_chart` — переход к секции «Таблицы сервера» с фильтрацией по этому серверу и автоматической прокруткой вниз |
| 🔗 | Иконка `open_in_new` — переход в Server Details для выбранного сервера |

В таблице также отображаются серверы, по которым ещё нет данных мониторинга (статус `no_data`, серые прочерки).

Рядом — график истории использования дисков (ECharts). Выбор периода: 30, 60, 90, 180 или 365 дней (dropdown).

### Таблицы сервера

Нижняя секция показывает крупнейшие таблицы выбранного сервера (top-30/50/100) с графиком изменения размеров во времени.

### Интерактивность графиков

- **Клик по строке таблицы** — фильтрует график, показывая только выбранный элемент. Повторный клик снимает фильтр
- **Show all / Hide all** — кнопки для управления видимостью всех серий
- **Легенда** — расположена справа от графика, клик по элементу включает/выключает серию
- **Изменение высоты** — потяните за нижний край графика (drag-to-resize). Установленная высота сохраняется между сессиями для каждого пользователя

## Просмотр таблиц

Центральная панель показывает все таблицы сервера (кроме системных баз данных). Для каждой таблицы отображается:

| Колонка | Описание |
|---------|----------|
| Table | Полное имя таблицы (база.таблица) |
| Size | Размер данных на диске |
| % | Процент размера таблицы от общего использования диска |
| R | Признак репликации (иконка синхронизации) |
| Refs | Количество ссылающихся объектов (MV, словари и др.) |
| _d | Количество Distributed-таблиц, ссылающихся на данную таблицу |
| Last SELECT | Время последнего SELECT-запроса |
| Last INSERT | Время последней вставки данных |
| TTL | Правила TTL таблицы (если настроены) |

### Фильтр по схемам

Над таблицей расположены кнопки-переключатели баз данных (schema). Нажмите на кнопку, чтобы показать/скрыть таблицы конкретной базы данных. Активные схемы выделены синим, скрытые — серым.

- **Clear all** (иконка корзины) — скрыть все схемы
- **Show all** (иконка select_all) — показать все схемы

### Текстовый фильтр

Под кнопками схем расположено поле поиска — введите фрагмент имени таблицы для мгновенной фильтрации списка. Кнопка `×` сбрасывает фильтр.

### Сортировка

Нажмите на заголовок колонки для сортировки. По умолчанию таблицы отсортированы по размеру (убывание). Заголовки таблиц фиксируются при прокрутке (sticky).

### Пагинация

Внизу таблицы можно выбрать количество строк на странице: 20, 50, 100, 200 или «All».

### Ссылки (Refs)

Если у таблицы есть ссылающиеся объекты - число в колонке Refs кликабельно. Нажмите, чтобы увидеть список (материализованные представления, словари и др.). Список можно скопировать.

### Экспорт данных

Под таблицей расположены кнопки экспорта:

- **CSV** (иконка download) — скачать данные в формате CSV
- **Excel** (иконка table_chart) — скачать данные в формате Excel (.xlsx) со стилизованными заголовками, авто-шириной колонок и авто-фильтром

Экспорт учитывает все наложенные фильтры (схема и текст), но выгружает все строки без ограничения пагинации. Имя файла включает имя подключения и временную метку.

Аналогичные кнопки экспорта доступны на вкладке **Columns** в панели деталей таблицы.

### Обновление

Кнопка «Refresh» под таблицей обновляет данные с сервера.

## Детали таблицы (правая панель)

Нажмите на строку таблицы - справа выедет панель деталей с тремя вкладками: **Columns**, **Query History**, **Flow**.

- Повторный клик по той же строке — скрывает панель
- Клик по другой строке — переключает на новую таблицу
- Кнопка `X` в правом верхнем углу панели — закрывает панель
- Клик по области таблиц (вне панели) — скрывает панель
- Стрелка у правого края экрана — открывает/скрывает панель

Ширину панели можно изменить перетаскиванием за левый край. Установленная ширина сохраняется в рамках сессии.

### Вкладка Columns

Показывает колонки выбранной таблицы:

| Колонка | Описание |
|---------|----------|
| Column | Имя колонки |
| Type | Тип данных ClickHouse (String, UInt64, DateTime и т.д.) |
| Codec | Кодек сжатия (LZ4, ZSTD и т.д.) |
| Size | Размер колонки на диске |
| % | Процент размера колонки: `% от таблицы / % от сервера` |
| Refs | Ссылки на колонку из других объектов |
| _d | Distributed-ссылки на колонку |

Таблица сортируется по размеру - сразу видно, какие колонки занимают больше всего места.

### Вкладка Query History

Показывает историю запросов к выбранной таблице из `system.query_log`. Вкладка загружается лениво — данные запрашиваются только при первом переключении на неё.

**Фильтры** (над таблицей, в три строки):

Строка 1 — **User**: кнопки-плитки по пользователям (переносятся на новую строку при большом количестве). Синяя = показывать, серая = скрыть. Иконка корзины — сбросить все, иконка select_all — выбрать все.

Строка 2 — **Kind**: аналогично по типу запроса (Select, Insert и т.д.). Иконка корзины — сбросить все, иконка select_all — выбрать все.

Строка 3 — управляющие элементы:
- **Limit** — количество выводимых запросов (50, 100, 200, 500, 1000)
- **Direct** — переключатель. Если включён (по умолчанию), показываются только запросы, в тексте которых напрямую упоминается имя таблицы. Если выключен — показываются все запросы, с дополнительной колонкой «Direct» (+/пусто)
- **`</>`** — показывает SQL-запрос, которым приложение получает историю
- **Refresh** — обновить данные с сервера

Фильтры User и Kind связаны: при выборе пользователя недоступные для него типы запросов блокируются, и наоборот. Фильтрация выполняется на стороне сервера.

**Текстовый фильтр** — дополнительный поиск по любому полю в загруженных данных.

**Просмотр полного запроса** — нажмите на строку в таблице запросов:

- Отформатированный SQL (ключевые слова UPPERCASE, ClickHouse-функции в camelCase)
- Подсветка имени текущей таблицы (жёлтый маркер), с переключателем
- Кнопка Copy для копирования

### Вкладка Flow

Показывает потоки данных через выбранную таблицу в виде интерактивных диаграмм. Три под-вкладки:

- **MV Flow** — цепочка материализованных представлений: откуда данные приходят и куда уходят (по DDL из `system.tables`)
- **Query Flow** — цепочка переливки данных по истории запросов (`system.query_log`)
- **Full Flow** — объединённая диаграмма из обоих источников

На диаграмме: прямоугольники — таблицы, параллелограммы — материализованные представления. Текущая таблица выделена синим.

**Навигация по диаграмме**:
- **Перетаскивание** — зажмите левую кнопку мыши и тащите для прокрутки
- **Масштабирование** — Ctrl + колёсико мыши для увеличения/уменьшения
- Кнопки `+` / `-` в правом верхнем углу для ручного управления масштабом
- Кнопка `?` (Help) — подсказка по управлению диаграммой
- Настройка масштаба сохраняется в рамках сессии

**Полноэкранный режим**: кнопка `⛶` (fullscreen) рядом с кнопками масштабирования открывает диаграмму во всплывающем окне на весь экран. В полноэкранном режиме:

- Диаграмма автоматически масштабируется, чтобы вписаться в окно
- Кнопки `+` / `-` для ручного изменения масштаба
- Кнопка **Fit** — вернуть автоматический масштаб
- Прокрутка колёсиком мыши изменяет масштаб

## Вкладка Users

Вкладка **Users** (рядом с Tables) показывает статистику по пользователям ClickHouse из `system.query_log`.

| Колонка | Описание |
|---------|----------|
| User | Имя пользователя |
| Queries | Общее количество запросов |
| Last Query | Время последнего запроса |
| Total Time (s) | Суммарное время выполнения |
| Read | Объём прочитанных данных |
| Written | Объём записанных данных |
| Peak Mem | Пиковое использование памяти |
| SELECTs | Количество SELECT-запросов |
| INSERTs | Количество INSERT-запросов |
| Other | Количество прочих запросов |

### Фильтр по пользователям

Над таблицей — кнопки-плитки по пользователям (аналогично фильтрам в Query History). Синяя = показывать, серая = скрыть.

- **Clear all** (иконка корзины) — сбросить все фильтры
- **Show all** (иконка select_all) — выбрать всех пользователей

### Сортировка

Все колонки сортируемые. Колонки Read, Written и Peak Mem сортируются по фактическому размеру в байтах (а не по отформатированной строке).

### Drill-down по пользователю

Нажмите на строку пользователя — справа откроется панель «User queries» с детализацией запросов. Доступны два режима:

- **All queries** — все запросы по отдельности (время, тип, длительность, статус, текст)
- **Grouped** — группировка по `normalized_query_hash`: одинаковые запросы с разными параметрами объединяются. Показывается количество вызовов, ошибки, суммарное время

Фильтры в drill-down:
- **Status**: All / OK / Errors
- **Kind**: All / SELECT / INSERT / CREATE / Other

Нажмите на строку запроса — откроется диалог с отформатированным SQL и кнопкой Copy.

## Вкладка Text Logs

Вкладка **Text Logs** (рядом с Tables и Users) показывает серверные логи ClickHouse из `system.text_log`. Отображаются записи уровней Fatal, Critical, Error и Warning за последние 2 недели. Вкладка загружается лениво — данные запрашиваются только при первом переключении.

### Summary-панель (левая часть)

Записи агрегируются по потоку (thread) и уровню:

| Колонка | Описание |
|---------|----------|
| Thread | Имя потока (кнопка — клик открывает детализацию) |
| Level | Уровень лога с цветным бейджем: Fatal (чёрный), Critical (фиолетовый), Error (красный), Warning (оранжевый) |
| Last Time | Время последней записи |
| Count | Количество записей |
| Message | Превью сообщения (первые 100 символов) |

### Detail-панель (правая часть)

Нажмите на имя потока — справа откроется панель с отдельными записями для этого потока:

- **Event time** — время с микросекундами
- **Thread name** — имя потока
- **Level** — цветной бейдж уровня
- **Query ID** — идентификатор запроса
- **Logger** — имя логгера
- **Message** — текст сообщения

Кнопки над таблицей позволяют переключать видимость колонок. Состояние сохраняется между сессиями.

Нажмите на иконку глаза (👁) у записи — откроется полноэкранный диалог с полным текстом сообщения и кнопкой Copy.

Кнопка **Close** скрывает detail-панель.

## Просмотр сгенерированных запросов

Кнопка `</>` под таблицей таблиц показывает SQL-запросы, которыми приложение получает данные. Запросы можно скопировать.

## Помощь

Кнопка `?` в правой части шапки открывает встроенную документацию. Доступны разделы:

- Руководство пользователя (эта страница)
- [Руководство администратора](admin-guide.md)
- [Релизы](releases.md) — история изменений
