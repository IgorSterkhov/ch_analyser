# Релизы

## v0.6 (март 2026)

### Управление подключениями
- Кнопка **Test** в диалоге подключения — проверка соединения перед сохранением, результат (успех/ошибка) во всплывающем окне
- Кнопка **Copy** — копирование подключения с очищенным паролем
- Подключения в Settings отображаются в сортируемой таблице (вместо карточек)
- При переименовании подключения данные мониторинга автоматически мигрируются в DuckDB

### All Servers — дашборд
- Неподключённые серверы отображаются в таблице дисков (статус `no_data`)
- Иконка `table_chart` в строке сервера — фильтрация секции «Таблицы сервера» по этому серверу с автоматической прокруткой вниз
- **Drag-to-resize** для графиков: изменение высоты перетаскиванием за нижний край, высота сохраняется между сессиями для каждого пользователя
- Фикс сортировки дат на графиках — ось X переведена на тип `time` (вместо `category`), даты всегда отображаются хронологически

### Экспорт данных
- **Экспорт в CSV и Excel** — кнопки экспорта на вкладках Tables и Columns. Выгрузка с учётом наложенных фильтров (схема, текст), без ограничения пагинации. Excel-файл включает стилизованные заголовки, авто-ширину колонок и авто-фильтр

### Server Details
- **Фильтр по схемам** — кнопки-переключатели баз данных (schema) над списком таблиц с кнопками Clear all / Show all
- **Drag-to-pan и zoom** в диаграммах Flow — перетаскивание для навигации, Ctrl+колёсико для масштабирования, кнопка Help с подсказками
- **Loading indicators** — индикаторы загрузки (спиннеры) для таблиц и вкладок правой панели
- **Авто-перезагрузка** — при смене подключения активная вкладка Users / Text Logs автоматически обновляется данными нового сервера
- **Query Flow** — фильтрация диаграммы: показываются только таблицы из вертикальной цепочки выбранной таблицы (BFS-фильтрация, аналогично MV Flow)

### Прочее
- **Тултипы** — все кнопки, вкладки, колонки таблиц и элементы навигации снабжены всплывающими подсказками
- Стилизация кнопок **Show all** / **Hide all**: единый стиль с рамкой, высота выровнена с dropdown-элементами
- Новая зависимость `openpyxl` для генерации Excel-файлов
- Фикс sticky-заголовков таблиц: динамический расчёт высоты через JS
- Фикс просмотрщика документации: корректный скролл через absolute positioning

## v0.5 (март 2026)

### Дашборд All Servers
- Новая страница мониторинга всех серверов (кнопка «All Servers» в шапке)
- Таблица дисков с цветовой индикацией статуса: OK (зелёный), Warning (жёлтый), Critical (красный)
- Графики использования дисков (ECharts) с историей за выбранный период
- Таблица и график размеров таблиц по выбранному серверу (top-N: 30, 50, 100)
- Интерактивность: клик по строке таблицы фильтрует график, повторный клик — снятие фильтра
- Кнопки **Show all** / **Hide all** в легенде графиков
- Выбор периода: dropdown 30, 60, 90, 180, 365 дней
- DuckDB для локального хранения метрик мониторинга

### Вкладка Text Logs
- Новая вкладка **Text Logs** — серверные логи из `system.text_log` (Fatal, Critical, Error, Warning за 2 недели)
- Summary-панель: агрегация по потокам (thread), цветные бейджи уровней, количество записей
- Detail-панель: клик по потоку — детализация с отдельными записями, просмотр полного сообщения
- Переключаемые колонки, состояние сохраняется между сессиями

### Таблицы и колонки
- Колонка **%** в таблице Tables — процент размера таблицы от общего использования диска
- Колонка **_d** в таблицах Tables и Columns — количество Distributed-ссылок на таблицу/колонку
- Колонка **%** во вкладке Columns — процент размера колонки: `% от таблицы / % от сервера`
- **Текстовый фильтр** над списком таблиц — мгновенная фильтрация по имени

### Вкладка Users
- Новая вкладка **Users** — статистика по пользователям из `system.query_log`
- Колонки: User, Queries, Last Query, Total Time (s), Read, Written, Peak Mem, SELECTs, INSERTs, Other
- Фильтр по пользователям (кнопки-плитки), кнопки **Clear all** / **Show all**
- Сортировка по всем колонкам; Read, Written, Peak Mem сортируются по байтам
- **Drill-down по пользователю**: клик по строке открывает правую панель с историей запросов
- Два режима: **All queries** (все запросы по отдельности) и **Grouped** (группировка по `normalized_query_hash` — одинаковые запросы с разными параметрами объединяются; показывается количество вызовов, ошибки, суммарное время)
- Фильтры в drill-down: Status (All / OK / Errors), Kind (All / SELECT / INSERT / CREATE / Other)
- Клик по запросу — диалог с отформатированным SQL

### Settings — табы и управление
- Диалог настроек переделан в три таба: **General**, **Connections**, **Monitoring**
- General: выбор плотности строк таблиц (Compact / Default / Comfortable)
- Monitoring (admin): пороги использования диска — Warning % и Critical %
- Кнопка **Stop Service** (admin) — остановка сервиса из UI с подтверждением

### Улучшения графиков
- Легенда перенесена вправо (без перекрытия данных)
- Кнопки **Show all** / **Hide all** для управления сериями
- Клик по строке таблицы фильтрует график, повторный клик — сброс
- Dropdown выбора дней вместо слайдера
- Улучшенные тултипы: перекрёстный курсор, прокручиваемый контент

### Кнопки «Show all» в фильтрах
- Рядом с каждой кнопкой «Clear all» добавлена кнопка «Show all» (иконка select_all)
- Работает в фильтрах Query History (User, Kind) и вкладки Users

### Query History — просмотр по клику
- Просмотр полного SQL-запроса теперь открывается по клику на строку таблицы (иконка глазика убрана)

### Sticky-заголовки таблиц
- Заголовки всех таблиц фиксируются при прокрутке

### Прочие исправления
- Фикс лоадеров: panel.clear() перед рендером контента
- Фикс перекрытия графика и легенды (grid.right 25%)
- Фикс f-string для INTERVAL в DuckDB-запросах

## v0.4 (февраль 2026)

### Query History — улучшенные фильтры
- Фильтры User и Kind разнесены в отдельные строки
- User-кнопки отображаются плиткой с автопереносом (удобно при большом количестве пользователей)
- Кнопка сброса (иконка корзины) у каждого фильтра
- Переключатель **Direct** — фильтрация только прямых запросов к таблице (по тексту запроса). По умолчанию включён. При выключении добавляется колонка «Direct»
- Кнопка просмотра SQL `</>` перенесена в строку управляющих элементов

### Ленивая загрузка вкладок
- Вкладки Query History и Flow загружаются только при первом переключении на них

### Исправление подсчёта Refs
- Refs теперь считает только случаи, когда таблица является **источником** данных для другой сущности
- MV, которые пишут В таблицу (TO table), больше не увеличивают счётчик Refs

### Flow — полноэкранный режим
- Кнопка развёртывания диаграммы на весь экран (рядом с кнопками масштабирования)
- Автоматическая подгонка масштаба: диаграмма вписывается в экран
- Кнопка **Fit** для возврата к оптимальному масштабу
- Масштабирование колёсиком мыши в полноэкранном режиме
- Кнопки `+` / `-` для ручного управления масштабом

### Прочие исправления
- Прогресс-бар диска: исправлено отображение длинных дробных чисел

## v0.3 (февраль 2026)

### Вкладка Flow — визуализация потоков данных
- Новая вкладка **Flow** в панели деталей таблицы
- **MV Flow** — цепочка материализованных представлений (по DDL из `system.tables`)
- **Query Flow** — потоки переливки по истории запросов (`system.query_log`)
- **Full Flow** — объединённая диаграмма
- Диаграммы Mermaid: таблицы = прямоугольники, MV = параллелограммы, текущая таблица выделена синим
- Масштабирование диаграмм кнопками `+` / `-`, масштаб сохраняется в сессии

### Правая панель Table Details — выдвижной drawer
- Панель деталей таблицы переделана в выдвижную правую панель (overlay)
- Открывается/закрывается кликом по строке таблицы или стрелкой у правого края
- Закрывается по `X`, по клику вне панели, повторному клику по той же строке
- Изменение ширины перетаскиванием за левый край, ширина сохраняется в сессии

### Серверная фильтрация истории запросов
- Фильтры User и Kind теперь работают на стороне сервера (GROUP BY)
- Кросс-фильтрация: при выборе пользователя недоступные типы запросов блокируются
- Выбор лимита вывода: 50, 100, 200, 500, 1000
- Кнопка Refresh для обновления данных

### SQL-форматирование
- Новый модуль `sql_format.py` — форматирование SQL с сохранением ClickHouse camelCase-функций
- Корректное сохранение регистра для системных колонок (type, user, table и др.)

### Кнопка Copy — поддержка HTTP
- Копирование в буфер обмена теперь работает на удалённых серверах без HTTPS (fallback через textarea)

### Прочее
- Колонка **TTL** в таблице таблиц
- Колонка **R** (репликация) и **Refs** (ссылки) в таблице таблиц
- Просмотр списка ссылающихся объектов (MV, словари) по клику на Refs
- Ссылки на уровне колонок во вкладке Columns
- Процент заполнения диска округляется до 1 знака после запятой
- Встроенная документация (кнопка `?` в шапке)
